Бот торгует. Система решает, что критично, а что подождёт.

Автоматический торговый бот на Bybit генерирует поток алертов и обращений. Я построила систему, которая сама отделяет срочное от рутинного, хранит данные централизованно и отвечает клиентам без участия человека.

TriggerWebhook / Form
ModelClaude Haiku 4.5
RouteIF: priority
ActionGmail / NocoDB
01 / Контекст

Ручная обработка не масштабируется

Bybit-бот торгует по трём парам (SOLUSDT, XRPUSDT, SUIUSDT), стратегия Martingale Long/Short. Все уведомления и обращения клиентов разбирались вручную — без фильтра приоритета и без единого хранилища данных.

// до автоматизации

  • Алерты бота (стоп-лосс, ошибка API) идут единым потоком в Telegram
  • Критичность оценивается на глаз, вручную
  • Обращения клиентов — без шаблона и без истории
  • Данные о сделках нигде не консолидированы

// цель

  • Алерты классифицируются по приоритету автоматически
  • Критичные события маршрутизируются мгновенно
  • Клиенты получают персональный ответ без задержки
  • Все сделки — в едином API-доступном хранилище
02 / Архитектура

Четыре сценария, один процесс

Каждый сценарий — рабочий n8n-воркфлоу, протестированный на реальных данных.

01

Приём заявок и первичное хранение

Form Trigger нормализует данные и пишет в NocoDB через Header Auth, параллельно уходит уведомление в Telegram.

TriggerForm Trigger
StorageNocoDB (REST API)
РезультатЗаявка не теряется
02

Работа с данными сделок через API

GET получает список сделок, POST добавляет новую запись — пара, стратегия, плечо, инвестиция, PnL, ROI.

MethodsGET / POST
Authxc-token Header Auth
РезультатЕдиная таблица для аналитики
03

ИИ-классификация алертов бота

Текст алерта уходит в Claude Haiku 4.5, модель возвращает структурированный JSON (priority, event_type, action), IF-нода маршрутизирует critical/high в «Срочно».

ModelOpenRouter · Claude Haiku 4.5
OutputСтрогий JSON
РезультатКритичное — сразу, рутина — в дайджест
04

Webhook → ИИ → персональный email

Заявка клиента → промпт с жёсткими ограничениями (без выдуманных цифр доходности) → Gmail OAuth2 отправляет письмо.

TriggerWebhook (POST)
DeliveryGmail OAuth2
РезультатОтвет без участия оператора
03 / Надёжность

Ошибки — это часть процесса, не то, что прячут

Реальные проблемы, найденные и устранённые в ходе разработки.

✗ JSON.parse() → SyntaxError: model wraps response in ```json fences ✓ fix strip markdown wrapper before parse
✗ HTTP Request Body → bad control character in JSON at position N ✓ fix switch to expression-object body instead of text template
✗ ECONNREFUSED n8n → nocodb: different Docker networks ✓ fix docker network connect n8n_default nocodb
✗ nginx serving static defaults instead of proxying to n8n ✓ fix removed conflicting sites-enabled/default
✗ API key exposed in plaintext chat ✓ fix key rotation recommended + docker inspect output sanitized before sharing

06 / Для работодателя

Почему этот кейс полезен бизнесу

Операционная задача

Фильтрация и маршрутизация входящего потока по приоритету — вместо ручного прочтения каждого сообщения.

Ценность для бизнеса

Секунды вместо «когда заметят» на критичных событиях; рутинные ответы — без участия сотрудников.

Где ещё применимо

Служба поддержки, e-commerce (возвраты/жалобы), логистика — любой поток, где нужно отделить срочное от рутинного.

Инфраструктурная база

Готовый HTTPS + OAuth2 слой — можно подключать любые сервисы, требующие безопасной авторизации, без повторной настройки.
